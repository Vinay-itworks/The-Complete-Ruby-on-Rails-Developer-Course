Rails file structure has

  • app -> Its holds the MVC
    • assets -> contain static file for front end grouped into folders
    • channels -> makes communication possible
    • controller -> all controllers file are stored They handle web request from the user
    • helper -> all helper function to assist the MVC are stored
    • job
    • mailers -> contains mail specific function for application
    • models -> contains models and data stored in application database
    • views -> contains display templates to fill data in application
  • bin -> contains rails scripts to start application and other scripts to setup and upgrade app
  • config -> configures application database, roots and more
  • db -> contains database schema and database migrations file
  • lib -> contains extended model for application
  • log -> contains application logs
  • public -> contains static file and compiled assets
  • test -> contains unit test
  • tmp -> contains temporary file like cache
  • vendor -> contains third party code like vendor gems
  • .gitignore
  • Gemfile -> all our application gem dependency declared *Imp as includes rails core gems among other gems
  • Gemfile.lock -> It holds the gems dependency - Including all versions for the application
  • README.md -> It is a brief instruction manual for application
  • Rakefile -> will locate and load tasks that can be run from the command line
  • config.ru -> contains Rake configurations for rake based services which are used to start the application
